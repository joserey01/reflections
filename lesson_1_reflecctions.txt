¿Cuáles crees que sean los prost y lo contras cuando hacer un commit de forma manual (Como los haces con Git), contra que los cambios se hagan de forma automática (Como google docs lo hace)?
Se tiene documentada todas las versiones que se tienen y es genera varios commit segun el numero de versiones. En google no se documenta.
¿Por qué creeque algunos sistemas de control de versiones permiten guardar los cambios multiples archivos en un commit, mientras otros como google docs manejan cada archivo de forma separada?
Es que fueron desarrollado para diferentes funciones y posiblemente se requiere registro de otros tipos de archivos que no se reflejan en el comentario.
